```markdown
 Семейный бюджет — десктопное приложение

Десктопное приложение для управления семейными финансами. Позволяет вести учёт доходов и расходов, планировать бюджет, отслеживать баланс, формировать отчёты и экспортировать данные в Excel. Поддерживает нескольких членов семьи с разными ролями: администратор, участник (клиент) и гость.

 Возможности

- Управление членами семьи (только администратор): добавление, редактирование, удаление, импорт/экспорт списка (JSON).
- Доходы: добавление, редактирование, удаление. Типы: зарплата, стипендия, пенсия, дополнительный заработок. Счета: основной счёт, накопления, заначка.
- Расходы: фактические и плановые. Категории: продукты, одежда, коммунальные услуги, связь, отдых, обучение, развлечения, лекарства, транспорт, другое.
- Баланс: автоматический расчёт текущего баланса с контролем неотрицательности (нельзя потратить больше, чем есть – при нулевом или отрицательном остатке расходы запрещены).
- Отчёты за период: детализированный отчёт по доходам и расходам.
- Экспорт в Excel: выгрузка отчёта за выбранный период в формате `.xlsx`.
- История операций:
  - Журнал аудита действий всех пользователей.
  - Динамика изменения баланса с привязкой к операциям.
- Ролевая модель:
  - `admin` — полный доступ, управление членами семьи, добавление операций за любого участника.
  - `client` — доступ только к своим операциям (привязывается к конкретному члену семьи).
  - `guest` — только просмотр отчётов и баланса, без возможности редактирования.
- Работа в офлайн-режиме (десктопное приложение с локальным сервером).

 Технологии

 Фронтенд
- Vue 3 (Composition API)
- Vue Router – маршрутизация
- Axios – HTTP-клиент
- Vite – сборка

 Десктопная обёртка
- Electron – упаковка веб-приложения в нативное приложение
- electron-builder – сборка установщиков

 Бэкенд
- Python 3.9+
- FastAPI – REST API
- PostgreSQL – база данных
- psycopg2 – драйвер
- bcrypt – хеширование пароля администратора
- openpyxl – генерация Excel-отчётов

 Тестирование (бэкенд)
- pytest
- TestClient (FastAPI)

 Требования к системе

- Операционная система: Windows, macOS, Linux
- Установленные компоненты:
  - Node.js (v16 или новее)
  - Python (3.9–3.12)
  - PostgreSQL (13 или новее)
  - npm / yarn

 Установка и настройка

 1. Клонирование репозитория

```bash
git clone https://github.com/your-repo/family-budget-desktop.git
cd family-budget-desktop
```

 2. Настройка базы данных PostgreSQL

- Убедитесь, что PostgreSQL запущен.
- Создайте базу данных (по умолчанию используется `family_budget`):

```sql
CREATE DATABASE family_budget;
```

- (Опционально) Создайте пользователя и выдайте права:

```sql
CREATE USER budget_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE family_budget TO budget_user;
```

- Настройте параметры подключения в файле `backend/backend.py` (переменная `DB_CONFIG`):

```python
DB_CONFIG = {
    "dbname": "family_budget",
    "user": "postgres",          # измените при необходимости
    "password": "123",           # измените
    "host": "localhost",
    "port": 5432,
}
```

 3. Установка зависимостей для бэкенда

Рекомендуется использовать виртуальное окружение:

```bash
cd backend
python -m venv venv
source venv/bin/activate   # Linux/macOS
# или
venv\Scripts\activate      # Windows

pip install -r requirements.txt
```

Если файла `requirements.txt` нет, установите вручную:

```bash
pip install fastapi uvicorn psycopg2-binary bcrypt openpyxl python-multipart pytest
```

 4. Установка зависимостей для фронтенда и Electron

В корневой папке проекта:

```bash
npm install
```

 Запуск в режиме разработки

 1. Запуск бэкенда

```bash
cd backend
python backend.py
# или с uvicorn напрямую:
uvicorn backend:app --reload --port 8000
```

Сервер будет доступен по адресу `http://127.0.0.1:8000`.  
Документация API: `http://127.0.0.1:8000/docs`.

 2. Запуск фронтенда (Vite)

В отдельном терминале из корня проекта:

```bash
npm run dev
```

Фронтенд поднимется на `http://localhost:5173`.

 3. Запуск Electron в режиме разработки (с автообновлением)

```bash
npm run electron:dev
```

Приложение откроется в окне Electron, будет автоматически подключаться к бэкенду на `127.0.0.1:8000`.  
Перед первым запуском убедитесь, что бэкенд уже запущен.

 Сборка production-версии

 Сборка фронтенда

```bash
npm run build
```
Результат (`dist`) будет использоваться Electron.

 Сборка десктопного приложения

```bash
npm run electron:build
```

Сборщик (`electron-builder`) создаст установщик для вашей ОС:
- Windows – `dist/Семейный бюджет Setup.exe`
- macOS – `dist/Семейный бюджет.dmg`
- Linux – `dist/Семейный бюджет.AppImage`

Перед сборкой убедитесь, что параметры в `package.json` (раздел `build`) настроены корректно для вашей платформы.

 Использование приложения

1. Запустите бэкенд (или приложение в собранном виде, где бэкенд запускается автоматически).
2. Войдите в систему:
   - Администратор: логин `admin`, пароль `admin` (пароль можно изменить, установив переменную окружения `ADMIN_PASSWORD_HASH` – хеш bcrypt).
   - Гость: вход без данных, только просмотр.
   - Активация сеанса члена семьи: выберите существующего члена семьи из списка (предварительно их должен добавить администратор).
3. На главной странице (`/dashboard`) добавляйте доходы и расходы, управляйте членами семьи (для админа).
4. Формируйте отчёты за период и экспортируйте в Excel (`/report`).
5. Просматривайте историю действий и динамику баланса (`/history`).

 Тестирование

Для бэкенда предусмотрены unit-тесты с использованием отдельной тестовой базы данных (`family_budget_test`).

 Настройка тестовой БД

Убедитесь, что PostgreSQL запущен. Параметры подключения указаны в `backend/test_backend.py` (`TEST_DB_CONFIG`). При необходимости измените их под ваше окружение.

 Запуск тестов

```bash
cd backend
pytest test_backend.py -v
```

Тесты проверяют:
- аутентификацию (админ, гость);
- CRUD доходов и расходов;
- контроль отрицательного баланса;
- корректность расчёта баланса и отчётов;
- ведение журнала аудита.
